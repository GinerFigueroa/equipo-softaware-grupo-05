ENTRADAS

Este proceso se lleva a cabo durante la planificación del sprint y ayuda a desglosar las historias de usuario en tareas más pequeñas y manejables para el sistema de gestión de citas. Las estimaciones proporcionan una base para la planificación efectiva, permitiendo al equipo establecer expectativas realistas sobre lo que se puede lograr en el tiempo asignado, optimizando así la gestión del negocio y la experiencia del cliente.

Lista de tareas

Historia 1: Registro de usuarios

Tarea 1.1: Diseñar la interfaz del formulario de registro, incluyendo opciones de registro por correo y usuario.

Detalles: Debe incluir validación en tiempo real y mensajes de error claros

Tarea 1.2: Configurar la base de datos para almacenar la información de los usuarios de manera segura.

Detalles: Implementar cifrado de contraseñas y protección de datos

Tarea 1.3: Implementar la funcionalidad de registro con correo y contraseña.

Detalles: Flujo completo desde formulario hasta confirmación

Tarea 1.4: Configurar la autenticación con gmail utilizando OAuth2.

Detalles: Integración con API de Google y manejo de tokens

Historia 2: Reserva por Instagram

Tarea 2.1: Crear el diseño de la sección de reserva con botón de enlace a WhatsApp.

Detalles: Diseño responsive que destaque el CTA principal

Tarea 2.2: Implementar el enlace al chat de Instagram.

Detalles: Validar apertura en app móvil si está instalada

Historia 3: Visor de citas

Tarea 3.1: Diseñar e implementar la interfaz del visor de citas.

Detalles: Vista calendario y lista, filtros por fecha/estado

Tarea 3.2: Implementar funcionalidad de recordatorios.

Detalles: Notificaciones push y por correo programables

Historia 4: Gestión avanzada

Tarea 4.1: Diseñar interfaz para registro de citas y empleados.

Detalles: Formularios administrativos con validación

Tarea 4.2: Implementar funcionalidad de registro.

Detalles: CRUD completo con historial de cambios

Tarea 4.3: Desarrollar asignación de citas a empleados.

Detalles: Lógica de asignación automática/manual

HERRAMIENTAS

Criterios de estimación

Utilizaremos Planning Poker con escala Fibonacci modificada para estimar el esfuerzo relativo de cada tarea:

Puntos Esfuerzo equivalente Tipo de tarea
1 2-4 horas Cambios mínimos o ajustes
2 1/2 día Tareas simples sin dependencias
3 1 día Tareas estándar
5 2-3 días Tareas con múltiples componentes
8 1 semana Tareas complejas con integraciones
13 2 semanas Épicas que requieren división

Flujo de estimación

  1. Preparación: El PO presenta cada historia y sus criterios de aceptación
  2. Desglose: El equipo divide historias en tareas técnicas
  3. Discusión: Debate sobre complejidades y dependencias
  4. Votación: Estimación simultánea con cartas Planning Poker
  5. Consenso: Discusión de discrepancias y votación final
  6. Registro: Documentación de estimaciones acordadas
