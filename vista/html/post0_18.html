ENTREGABLES ACEPTADOS

Historia de Usuario Criterios de aceptación/Entregables ACEPTADO RECHAZADO
Historia 1: Registro de Usuario 1. El formulario permite registrar por correo y contraseña.
2. Una vez registrado, el usuario recibe confirmación visible de que el proceso ha sido exitoso y se le redirige a la página de inicio. 		X
Historia 2: Reserva de Cita 1. El enlace para reservar cita enfacebook debe estar ubicado de manera destacada en la página de inicio y ser claramente accesible.
2. Al hacer clic, el enlace se redirige correctamente al chat de whatsapp de la empresa sin interrupciones.
3. La dueña puede registrar manualmente las citas en el sistema una vez confirmadas con el cliente en el chat. 		X
Historia 3: Visor de Citas 1. El visor de citas debe mostrar claramente los detalles de cada cita.
2. El sistema permite al administrador ver las citas en tiempo real. 		X
Historia 4: Registro de Citas y Empleados 1. El sistema permite registrar mediante un formulario la cita del usuario.
2. Los empleados nuevos van a poder ser registrados de manera exitosa en el sistema por la dueña. 		X

SALIDAS

Acuerdo de entregables funcionables

El equipo Scrum y los stakeholders acuerdan que los siguientes entregables cumplen con los criterios de aceptación definidos y están listos para su implementación:

  • Sistema de registro de usuarios con opción de Google
  • Integración funcional con Instagram para reserva de citas
  • Visor de citas para administración del negocio
  • Formularios para registro de citas y empleados
