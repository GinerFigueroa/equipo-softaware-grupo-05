ENTRADAS

La estimación de historias de usuario es fundamental en la metodología Scrum para el sistema odontológico ZAZDENT, ya que permite cuantificar el esfuerzo requerido para implementar cada funcionalidad. Este proceso ayuda al equipo a planificar sprints realistas y gestionar adecuadamente el backlog del producto.

Historias de Usuario a Estimar

Historia 1: Agendar una cita

Como paciente, quiero agendar una cita desde la página web, para reservar una atención odontológica fácilmente.

Historia 2: Ver los tratamientos disponibles

Como visitante del sitio, quiero ver una lista de los tratamientos dentales ofrecidos, para saber qué servicios puedo recibir en la clínica.

Historia 3: Conocer al equipo odontológico

Como paciente nuevo, quiero ver los perfiles del personal de salud, para sentir confianza con quienes me atenderán.

Historia 4: Reportes financieros

Como propietario, quiero acceder a reportes financieros y de productividad, para tomar decisiones estratégicas sobre el consultorio.

HERRAMIENTAS

Métodos de Estimación

Para estimar las historias de usuario del sistema odontológico, el equipo utilizará el método Planning Poker, una técnica colaborativa que fomenta la discusión y el consenso sobre el esfuerzo requerido para cada historia.

El equipo utilizará la siguiente escala de Fibonacci modificada para las estimaciones:

  • 1 punto - Muy sencillo (ej. cambios menores)
  • 3 puntos - Sencillo (ej. componente simple)
  • 5 puntos - Moderado (ej. componente con integraciones)
  • 8 puntos - Complejo (ej. múltiples componentes)
  • 13 puntos - Muy complejo (ej. requiere investigación)
