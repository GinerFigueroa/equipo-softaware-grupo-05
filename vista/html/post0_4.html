ENTRADAS

Desarrollar épicas en la metodología Scrum es crucial para ZAZDENT Estudio, ya que permite tener una visión global del proyecto odontológico y descomponer las grandes funcionalidades en historias de usuario más manejables. Esto facilita la priorización de las características que aportarán mayor valor al negocio, asegurando que el equipo trabaje en lo más relevante primero. Además, las épicas fomentan la colaboración entre el Product Owner, los stakeholders y el grupo de Scrum, alineando sus expectativas y objetivos.

HERRAMIENTAS

Reuniones del grupo

El Product Owner se reúne periódicamente con los dueños del negocio para recabar sus necesidades, informarles sobre la página web y aclarar dudas. Estas reuniones se realizan de forma virtual mediante aplicaciones como Google Meet, para la comodidad de los stakeholders. Solo se procede a reunir con los dueños del negocio, ya que ellos son las partes interesadas con mayor poder en el negocio.

SALIDAS

Épicas

Las épicas son esenciales porque proporcionan una visión general de las funcionalidades del producto, ayudan a priorizar tareas y permiten descomponer el trabajo en historias de usuario más manejables. Fomentan la colaboración entre el equipo y los stakeholders, ofrecen flexibilidad ante cambios en los requisitos y facilitan el seguimiento del progreso, asegurando que el trabajo esté alineado con los objetivos estratégicos de la organización y maximizando el impacto de las entregas.

Épica 1: Página de bienvenida

Descripción: El sistema debe presentar una página de bienvenida cuando se ingrese al sitio web, funcionando como una “landing page” atractiva visualmente, con fácil acceso a las principales secciones y manteniendo una carga rápida.

Criterios de éxito:
  • Diseño Visual Atractivo: La página debe contar con colores vibrantes, imágenes de alta calidad y videos integrados para captar la atención del usuario.
  • Carga Ágil: A pesar del contenido visual, la página debe estar optimizada para cargar rápidamente en diferentes dispositivos y conexiones a internet, asegurando una buena experiencia de usuario.
  • Facilidad de Navegación: La interfaz debe ser sencilla y amigable, permitiendo al usuario navegar de manera intuitiva.
  • Footer: Incluir columnas con el logo, horario de atención, enlaces a redes sociales y secciones clave.
  • Barra de Navegación: Debe incluir secciones como Sobre nosotros, Servicios, Precios, Ubícanos y Reservas.

Épica 2: Gestión de Citas de Clientes

Descripción: El sistema debe permitir la gestión eficiente de citas odontológicas, proporcionando información de horarios y días disponibles.

Criterios de éxito:
  • Los clientes pueden reservar citas en línea.
  • El sistema genera recordatorios automáticos vía email o SMS.
  • El sistema refleja la disponibilidad en tiempo real, evitando reservas duplicadas.
  • Sección de administrador donde se visualizan todas las citas pendientes y atendidas.

Épica 3: Gestión de contenido del sitio Web

Descripción: El sistema debe permitir agregar o modificar contenido visible en el sitio web como un CMS.

Criterios de éxito:
  • Facilidad de Uso: Los administradores pueden acceder a un panel de control para editar y actualizar las secciones del sitio web.
  • Gestión de Promociones: Los administradores pueden actualizar promociones de forma ágil.
  • Previsualización de Cambios: Antes de publicar, el CMS debe ofrecer una opción de previsualización.
